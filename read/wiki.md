# ATMEGA328P EXECUTOR - РУКОВОДСТВО

## 1. ЧТО ЭТО?

Система для микроконтроллера ATmega328P.
Управляет светодиодами (10×10), звуком, клавиатурой.
Программы пишутся числами и отправляются через USB.

---

## 2. ФАЙЛЫ СИСТЕМЫ

| Файл | Что делает |
|------|-----------|
| led.asm | Главный исполнитель команд |
| name.asm | Номера светодиодов (0-99) |
| audio.asm | Управление звуком |

---

## 3. КАК РАБОТАЕТ ИСПОЛНИТЕЛЬ (led.asm)

### Главный цикл:
1. Включиться
2. Настроить пины
3. Спросить имя файла (или сразу ждать программу)
4. Загрузить программу через UART
5. Выполнять команды по очереди
6. Закончить

### Что такое программа?
Это просто список чисел (байтов).
Каждое число = команда или данные.

**Пример:** `01 0A 05 64 00 02 0A FF`

---

## 4. ТАБЛИЦА КОМАНД

| Код | Название | Аргументы | Действие |
|-----|----------|-----------|----------|
| 01 | led-on | 1 байт (0-99) | Включить светодиод |
| 02 | led-off | 1 байт (0-99) | Выключить светодиод |
| 03 | rdg.key | нет | Ждать нажатия клавиши |
| 04 | cnd | 1 байт (ASCII) | Условие: если клавиша совпадает → пропустить 2 байта |
| 05 | delay | 2 байта (мс) | Задержка в миллисекундах |
| 06 | audio-on | нет | Включить звук (440 Гц) |
| 07 | audio-off | нет | Выключить звук |
| FF | end | нет | Конец программы |

---

## 5. КАК ПИСАТЬ ПРОГРАММУ

### Правила:
- Каждая команда = 1 байт
- Аргументы идут сразу после команды
- В конце обязательно `FF` (конец программы)

### Пример 1: Зажечь светодиод #10 на 100 мс
 01 0A 05 64 00 02 0A FF
